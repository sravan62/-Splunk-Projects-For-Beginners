# -Splunk-Projects-For-Beginners
A collection of beginner-friendly projects focused on learning and practicing log analysis using Splunk. This repository covers real-world SOC use cases such as DNS log analysis, network traffic monitoring, detecting suspicious activities, and basic threat hunting techniques.

## Projects

1. [DNS Log Analysis with Splunk](https://github.com/sravan62/-Splunk-Projects-For-Beginners/blob/main/DNS%20Log%20Analysis%20with%20Splunk.md) : This project demonstrates a structured approach to analyzing DNS (Domain Name System) logs using Splunk SIEM. It includes steps for ingesting log data, extracting key fields, and performing analysis to identify DNS query trends. The project also focuses on detecting suspicious activities, uncovering anomalies, and monitoring DNS traffic to support effective threat detection.

2. FTP Log Analysis with Splunk : This project demonstrates a structured approach to analyzing FTP (File Transfer Protocol) logs using Splunk SIEM. It includes steps for ingesting log data, extracting key fields such as source IP, destination IP, username, file actions, and transfer status. The project focuses on identifying anomalous file transfers, detecting unauthorized access, and monitoring FTP activity to support effective threat detection and incident response.

3. SSH Log Analysis with Splunk : This project demonstrates a structured approach to analyzing SSH (Secure Shell) logs using Splunk SIEM. It includes steps for ingesting log data, extracting key fields like source IP, destination IP, authentication attempts, and session durations. The project highlights techniques to detect brute-force attacks, unauthorized access attempts, and suspicious login behavior, providing actionable insights for enhancing system security.

4. DHCP Log Analysis with Splunk : This project demonstrates a structured approach to analyzing DHCP (Dynamic Host Configuration Protocol) logs using Splunk SIEM. It includes steps for ingesting log data, extracting key fields such as MAC addresses, assigned IPs, lease times, and client IPs. The project emphasizes detecting unauthorized device connections, IP conflicts, rogue DHCP servers, and abnormal network behavior to support proactive threat detection.
